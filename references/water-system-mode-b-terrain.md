# Water System Mode B Terrain

## 水系图制作规则（地形分层设色模式）

底图为 DEM 高程分层设色 + 山体阴影，河网与流域全部由 DEM 水文分析导出。工具集位于 `OneNote/户外旅行/水系/scripts/terrain_dem/`（含配好的 Python venv）。参考成品：北盘江、赣江、湘江水系图（`scripts/outputs/北盘江水系图_地形渲染.png`、`scripts/outputs/赣江水系图_地形渲染.png`、`scripts/outputs/湘江水系图_地形渲染.png`）。

- **数据来源**：高程用 AWS Terrain Tiles（Terrarium 编码 PNG 瓦片，`https://s3.amazonaws.com/elevation-tiles-prod/terrarium/{z}/{x}/{y}.png`，通常 z10；解码 `elev = R*256 + G + B/256 - 32768` 米）；河名/城镇/水电站/山峰/省界用 OpenStreetMap（Overpass API）。图上须注明来源并声明不能替代工程测量或勘界。
- **河名与命名种子来源**：模式 B 的河网几何来自 DEM，但河流名称和命名通道种子不得靠目测或版面估计。每条重要命名支流必须先从 OSM `waterway=*` 且 `name`/`name:zh` 匹配的河道、地方政府/水利部门资料、地名志、百科或可靠地图资料中确认河名、流向和大致位置；优先采用 OSM 同名河道几何上的上游端、中游稳定段或多个控制点作为 D8 追踪种子，OSM 缺失时才可使用公开资料给出的源头、流经乡镇、汇入口或坐标，并记录 URL、查询日期、资料口径和置信度。配置中不得只写裸经纬度；`named_river_sources` 或同类结构应至少包含 `name`、`seed_lonlat`、`source`/`source_id`/`source_url`、`verified_at`、`confidence`，可选 `waypoints`、`mouth_hint`、`aliases`。手工只允许调整文字锚点和偏移，不得作为河流身份或种子点依据。
- **Overpass 端点**：主端点 `overpass-api.de` 常返回 406/504，备用 `https://lz4.overpass-api.de/api/interpreter` 更稳；大范围 `admin_level=4` 省界须用 `[bbox:...]` 全局裁剪，否则整省几何超时；中文名用正则 `["name"~"关键字"]` 匹配。
- **流域划分（pysheds，核心）**：`fill_pits → fill_depressions → resolve_flats → flowdir(D8) → accumulation → snap_to_mask 吸附出水口 → catchment`。出水口须选在**干流汇合口上游约 2 km 的目标干流上**，避免把相邻水系并入。校验：北盘江算得约 2.83 万 km²，官方约 2.66 万 km²，吻合（DEM 分辨率略大属正常）。水文分析用 F=2 降采样（约 150 m/格），numba 加速数秒完成。
- **河网必须从 DEM 提取，不叠加 OSM 河流**：OSM 河流上游常标为 `stream` 而缺失、分段有缺口；用 `extract_river_network` 从流量累积提取的河网连续、无缺口、延伸至分水岭。OSM 河流仅用于放置河名标签。两套几何叠加会造成线条粗细/颜色突变，严禁并绘。
- **河网默认统一色相、线宽只随流量连续变化**：模式 B 的各级支流默认使用同一种浅蓝色（湘江支流采用 `#56B4E9`），不再按集水面积改变颜色；每条河段仍附 `acc_km2`，线宽按其对数连续映射（约 120–8000 km²：0.58→2.36px），避免离散阈值处突然变粗。干流不得另加固定宽度，必须与支流调用完全相同的流量—线宽函数，并按各河段实际集水面积沿程自然变粗。若白色河谷背景使干流显得过浅，单图可保持色相和饱和度不变，仅降低干流 HSL 明度；湘江为明确例外：支流 `#56B4E9`，干流 `#12618D`，即 HSL 明度降低 50%，但线宽规则完全相同。
- **干流周围的白色渐变背景不是白线描边**：参考地形图的处理是在干流周围铺一层可辨认但边缘柔和的透明白色背景，用于提亮河谷，而不是沿干流画白色宽线或白边。实现时应把干流中心线栅格化，再按到中心线的距离生成连续高斯透明度场；湘江最终采用中心最大 α=0.45、σ=8 px，白色更集中、扩散范围较窄，并向外平滑衰减至 0。白色背景层先绘制，随后绘制浅蓝支流，再按相同流量—线宽函数叠绘深蓝干流。验收必须加入关闭渐变的 A/B 对照，确保整图比例下肉眼可见；禁止用少数几条不同宽度的半透明白线叠加，以免出现套圈或分层带。
- **干流与公认河源追踪必须用命名控制走廊约束**：先从出水口沿流向场逐格向上游追踪（每步取流量累积最大的上游格）得到原始主河道；公认源头附近往往位于分水岭，DEM 降采样后相邻一格就可能进入另一水系，因此不能把单一源头坐标直接当作无条件种子，也不能以“源点候选到原始干流的连接段最长”选择源流。正确做法是在源点小半径邻域枚举候选格，沿 D8 下行并与原始干流拼接，要求拼接后的完整干流依次通过权威源流的多个下游控制点，再按距官方源点最近、保留原始干流段更长的顺序选择。湘江须通过大桥河—潇水方向及道县、双牌、永州控制走廊；2026-07-06 修复前的最长连接段算法误选舜水—钟水—舂陵水一线，已禁止复用。河源山两侧常为相邻水系共同发源地（如马雄山同为北盘江与南盘江源），必须验证候选格的实际下游归属。
- **河名标签必须绑定专属河道**：手填文字锚点只能控制版面位置，不能证明河流身份；不得在全部 DEM 河网中简单选择离锚点最近的河段。重要命名河流应从经 OSM 或公开资料核实的源区/中游种子沿 D8 追踪到干流，必要时要求路径通过公开资料中的流经乡镇、汇入口或 OSM 同名几何重合控制点；追踪结果保存为命名通道，标签只能在该通道上吸附。专属通道缺失、与 OSM/公开资料几何重合度过低、汇入口不符、距文字锚点超限或下游未接入目标干流时，必须报错或标为待核实，不能静默落到邻河。湘江图的“舂陵水”已绑定舜水—钟水—舂陵水 D8 通道。
- **格网中心坐标转换必须可逆**：若 `row_col_to_lon_lat()` 用 `(row+0.5, col+0.5)` 输出格网中心，逆变换必须用包含格的 `floor` 规则，不能对半格坐标使用 `round()`；否则会读取相邻格的累计面积，使出口面积从约 9.37 万 km² 错成单格面积。所有水文脚本必须有“格网索引→中心经纬度→原格网索引”的往返测试，并验证干流累计面积从源头向出口单调不减。
- **分层设色与晕渲**：色带绿→黄→橙褐映射流域高程区间（北盘江用 150–2600 m）；山体阴影方位 315°、高度角 45°、`vert_exag≈2.2`，阴影强度保留 0.55–1.0 以免压暗设色。
- **流域高亮**：流域外叠加半透明深色（约 `(0.08,0.09,0.12,α=0.40)`）做聚光灯效果，流域内保持明亮；流域边界红实线（`#8e1b1b` 2.6px）+ 白描边双线。
- **要素与文字规范同通用规则**：省界紫色虚线 + 半透明省名；地级市红大圆点大字、县城灰白小圆点小字；水电站紫三角、分水岭山峰褐三角+海拔、干流汇口红星；山脉名红色书法体沿走向旋转。**山脉名称须逐字沿山系起止点插值排布，并明显拉开字距，尽量覆盖所指山脉的完整延伸区域；不得只在单一锚点写成紧凑词组。**文字须逐一检查不与河名/彼此重叠，拥挤处宁可删次要标注也不叠字。经纬网、比例尺、指北针、图例齐全；标题用简洁名（如「北盘江水系图」），不写方法细节。
- **比例尺应服从版面空间**：不得为保留较大的整百公里比例尺而占据大片地图区域。比例尺可作为图例最后一项放入左下角或右下角图例框，图例位置按标注密度选择，并根据图例可用宽度选择 10、20 等易读整数；显示线段仍须在最终版式完成后按地图坐标变换计算，不能退化为无真实尺度的装饰线。湘江图采用图例内嵌 10 km 比例尺，原左下角 100 km 独立比例尺已删除。
- **环境**：macOS Homebrew Python 为 PEP 668 外部管理，须用虚拟环境（`terrain_dem/venv/` 已配好）。关键包 `pysheds 0.5`、`numba`、`rasterio`、`scipy`、`matplotlib`、`contextily`。pysheds 0.5 用了 numpy 2.0 已移除的 `np.in1d`，脚本开头须加 `if not hasattr(np,'in1d'): np.in1d = np.isin`。可用 `pip install -r terrain_dem/requirements.txt` 重建。
- **平原分汊河口的出口选择**：若目标河流在入湖前分汊，不能把 catchment 出口直接放在某一条汊道上，否则 DEM 可能只圈出局部流域。湘江濠河口即属此例：直接吸附后仅得到约 3.06 万 km²；将计算出口前移到分汊前的望城乔口控制点（112.748°E、28.5148°N）后得到约 9.37 万 km²，与普查 9.47 万 km²吻合。成图仍可把实际入湖口作为河口节点标出。
- **运行**：`cd scripts/terrain_dem && ./venv/bin/python fetch_dem.py`（换区域时改经纬范围）`→ watershed_pysheds.py`（改出水口坐标）`→ 对应流域渲染器`。现有多流域配置集中在 `terrain_runtime.py`；设置 `WATERSHED_NAME=ganjiang` 或 `xiangjiang` 可分别在 `runs/ganjiang/`、`runs/xiangjiang/` 生成隔离的中间数据，再运行 `render_ganjiang.py` 或 `render_xiangjiang.py` 出图，避免覆盖其他流域缓存。成品统一输出到 `scripts/outputs/`。
