# Water System Mode A Satellite

## 水系图制作规则（卫星底图模式）

以后制作北江、东江或其他类似水系图，默认遵循以下规范：

- 数据分工：HydroBASINS 用于流域范围，HydroRIVERS 用于连续河网、`ORD_STRA` 级别和 `NEXT_DOWN` 拓扑，OpenStreetMap 用于河名、县城，卫星影像可使用 Esri World Imagery。图上必须注明来源，并声明不能替代工程测量、水利管理或行政勘界图。
- **中国大陆网络环境替代方案**：
  - **省界数据**：OSM Nominatim 在中国大陆不可达时，可改用阿里云 DataV GeoJSON API（`https://geo.datav.aliyun.com/areas_v3/bound/{adcode}_full.json`）。贵州 adcode=520000，重庆=500000，四川=510000，湖南=430000，湖北=420000。数据包含各市县级边界，需用 `unary_union` 合并后按地图范围裁剪。复杂几何需先 `make_valid()` 再合并。
  - **卫星底图（含坐标系关键坑）**：优先使用 Esri World Imagery（`https://services.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}`）。**Esri 是 WGS-84，与 HydroRIVERS/OSM 河网同坐标系，矢量与影像天然对齐**。⚠️ **高德/webst 卫星瓦片是 GCJ-02（火星坐标），与 WGS-84 河网存在约 500m 的非线性偏移**：全流域尺度看不出，但放大到几公里就会出现"河爬到湖面、河不贴谷"的严重错位。因此**默认必须用 Esri**；只有 Esri 不可达且愿意接受偏移、或已实现 WGS84→GCJ-02 纠偏（对所有河网/流域界/标注坐标先转 GCJ-02 再投影）时，才可退用高德。2026-07-06 已将乌江、韩江、榕江、练江、渭河、汾河和南盘江统一切换为 Esri 并重新出图；当前北盘江成品为 DEM 地形版，不使用卫星底图。
  - **卫星层级必须按流域面积和地图范围选择**，不得把上一幅图的 zoom 机械复制到下一幅图。建议初始值：流域小于 2,000 km² 用 z12；2,000–10,000 km² 用 z11；10,000–50,000 km² 用 z10；50,000–150,000 km² 用 z9；更大流域用 z7–8。面积只用于初选，最终还必须按实际 `MAP_BOUNDS` 计算原生瓦片拼图尺寸。
  - **底图清晰度验收**：卫星拼图在裁剪后、送入 Matplotlib 之前，横向原生像素原则上不得低于 2,500 px，且不得低于最终 PNG 横向像素的约 65%；若不满足，应提高 zoom 或缩小地图范围，不能依赖 `imshow` 放大模糊底图。提高层级后必须目检山脊、城市纹理和海岸线是否清晰，同时避免下载超出地图范围的无关瓦片。练江约 1,353 km²，采用 z12。
  - **OpenStreetMap Overpass API**：在中国大陆也不可达，但河名查询结果可缓存复用。首次运行需在可达网络环境下预下载 `osm_named_rivers.json` 缓存。
- 河网连续性优先于命名几何：OSM 同一河名常由多个断开的 way 组成，只用于识别河名和放置标签；高亮河道应沿 HydroRIVERS 拓扑绘制。遇到上游断裂时，从命名几何最上游端点选择最近的水文河段作为种子，再沿 `NEXT_DOWN` 单一路径追踪到干流；不得用直线硬连，也不得把附近侧支沟一并加粗。
- **干流必须按 `NEXT_DOWN` 单路径追踪，禁止用距离缓冲识别**：不得用"距 OSM 干流线某距离（如 0.004°）以内的河段都算干流"的写法——该缓冲会把干流附近所有小汇口短河段一并纳入并画成粗线，形成大量"毛刺"。正确做法是取最接近源点的水文河段为种子，沿 `NEXT_DOWN` 逐段追踪到流域出口，得到唯一主河道（见南盘江 `trace_mainstem_ids()` 与 config `SOURCE_POINT`）。**毛刺问题九成出在这里，先查干流识别方式再怀疑支流阈值。**
- **主要支流走廊的"停止条件"也必须用拓扑而非距离**：走廊从支流种子沿 `NEXT_DOWN` 追踪、遇到干流即停，停止集必须是**干流河段 ID 集合 `mainstem_ids`**，不能用"距干流几何某距离以内"来判停——距离判停会把支流自己那段紧邻汇口的河段也当成停止段排除掉，导致**支流末端接不到干流、出现断口**（尤其背景河网提到 ORD≥4、本可桥接的小段被隐去后断口暴露）。见南盘江 `corridor_ids_for_geometry(..., mainstem_ids)`。凡"支流未连到干流"的断口，先查此处。
- 河线粗细采用 Strahler 级别连续缩放：低级支沟约 0.35–0.65，次级河流约 0.75–1.35，主要支流约 1.5–2.8，干流主轴约 3.0–4.6。同一河流应随汇流自然变粗，不得把整条命名河流画成恒定粗线。干流、主要支流、次级河网使用同一青色系，以线宽、明度和字重区分。
- **河网默认只显示到三级支流**：背景河网默认仅绘制 HydroRIVERS `ORD_STRA >= 3` 的河段，`ORD_STRA` 1–2 的毛细支沟隐去；干流和已经核实的主要支流为保持源头至汇入口连续，可沿 `NEXT_DOWN` 拓扑保留局部低于三级的河段。若专题图确需展示小流域、城市内河或源区沟谷，必须在单图配置中明确覆盖此默认值。**实现时须真正跳过（`continue`）被隐去河段，不得只调低透明度画出全部河网。****大流域（Strahler 阶数高、面积数万 km² 以上，如南盘江）中 `ORD_STRA==3` 的三级小流仍有数百条、观感杂乱，此时背景河网起绘阶数应上调到 `ORD_STRA >= 4`（config 常量 `SECONDARY_MIN_ORD`），干流与已命名主要支流走廊不受此限、照常保留，既清爽又不丢重要支流。**
- **短支流系统按连通总长再过滤**：满足三级阈值后，背景次级支流里仍会有很短的"尖刺"影响观感，须再按**连通支流系统总长**隐去。做法：把次级河段（`ORD_STRA>=3` 且不在干流/主要支流走廊内）通过 `NEXT_DOWN` 归并成连通支流系统（并查集），整条系统总长小于阈值才隐去；**必须先归并再量长，不能按单条河段长度过滤，否则会把由多段短河段组成的长支流整条误删，也会在河中间产生断口**。默认阈值约 `0.08°`（≈9 km），在单图 config 中以常量给出（如南盘江 `MIN_SECONDARY_TRIB_LEN_DEG`），可按流域大小和观感调整；干流与主要支流走廊永不参与此过滤。
- 省界与流域界必须区分：省界使用具有独立颜色的暖黄、橙褐或其他低饱和度彩色虚线，流域界用浅青白实线；不得再用容易与流域界混淆的灰白虚线。行政区多边形必须先取真实边界再按地图范围裁剪，避免把裁剪框误画成省界；省界不得遮盖主河道。
- **国境线与省界必须区分（三种模式通用）**：图幅涉及国境线或跨境水系时，国境线不得沿用省界虚线、流域界实线或普通行政边界颜色；应采用独立的高识别线型（如红色点划线、红白点划线或黑白边界线），图层高于省界但不得遮盖主河道和关键注记。国境线必须优先使用 OSM `admin_level=2` 国界关系或 Natural Earth `admin_0_boundary_lines_land` 等专门国界线数据；省级/县级行政区外缘、DataV 省界外缘或邻省合并外边界不得替代国界数据。若 OSM 与 Natural Earth 在局部明显冲突，须在脚本或说明中记录比较结果、取舍理由和查询日期。图例必须单列“国境线”，配套笔记须说明其数据来源与口径。
- 江名、水名和水库名统一使用亮蓝色，干流仅通过字号、粗体和描边突出，不使用黄色河名。河名锚点必须吸附到对应河线或经确认的水文通道上。
- 城镇符号必须按行政层级区分：地级市使用大双圈靶心，县级市及市辖区使用中等空心双圈，县城使用小实心圆点，源头继续使用菱形。城镇点图层必须高于河名图层，确保佛冈等点位不会被河名遮住。图例不是强制元素；当线型、颜色和点符号已经直观且版面空间有限时，可省略简单图例，把空间让给水系、山脉和地理节点。
- 乡镇默认不作为行政驻地标注，不能套用县城的圆点、字号和字体层级。若某个乡镇位置同时是汇流口、源头转换、古港、河口、峡谷入口等关键地理节点，应按“地理节点”而不是“乡镇”处理，使用独立的符号、颜色和字体体系；例如双溪咀应与县城明显区分。
- 流域行政范围涉及的地级市必须全部标出：先用流域边界与地级行政区边界求交，建立完整地级市清单，再标注各地级市驻地；不得只选择干流沿线或主观认为重要的地级市。县级市、市辖区和县城可按水系关系与版面密度筛选。
- 城市与县城点不得因文字避让而移动真实位置，文字可使用单独的经纬度偏移，但偏移距离必须保持文字与点位关系清楚，不得为避让而把县名推到远离标注点的位置。源头、重要城市、普通县城、关键地理节点、水库和省名使用不同字号或颜色层级。
- 全流域图除目标流域河网外，还应绘出地图范围内与其分水、邻接或共同进入同一平原/河口区域的相邻水系干流，并采用低于目标流域的视觉层级标注名称，以明确水系关系；例如榕江图应同时显示韩江、练江。相邻干流可用 HydroRIVERS 拓扑或经核实的 OSM 线形，但不得画成无数据依据的示意直线。
- **山脉与山峰数据来源规范（三种模式通用）**：为表现山脉与河流走向关系，全流域水系图可精选约 5-10 条主要山脉和 5-10 座代表峰，尤其应包含流域分水岭山脉。山脉名使用红紫色并按山势方向旋转；山峰使用红紫边白心三角形，标注峰名和海拔。山峰坐标与海拔必须优先查询 OSM `natural=peak` 节点（含 `ele`、`name`、`name:zh`、节点 ID）和地方政府、自然保护地、地名志、百科等公开资料；采用网页资料时必须记录 URL、查询日期和资料口径。若 OSM 与网页资料冲突，应优先采用有明确坐标/海拔且来源较权威者，并在配置或配套笔记中说明差异；若只能确认山峰存在但坐标不稳，不得标为精确峰点，可降级为“约位置”或仅保留山脉名。配置中不得再只写裸坐标；重要峰点应记录 `source`、`source_id` 或 `source_url`、`verified_at`、`confidence`，并尽量加入自动测试：校验峰点在地图范围内、距对应 OSM/资料坐标在可接受阈值内、位于相关分水岭/山系侧而非河谷或相邻水系错误一侧。山脉控制线同样不能凭版面感觉估算，应依据 DEM 分水岭、OSM 山峰/山脊点、地方政府地理描述和与主要河流的相对位置校核；如哀牢山必须位于元江/红河西侧这类关系，应写入回归测试。
- **地名、县城名、山脉名、山峰名、水库名、水利设施名和省名不得与河名文字或彼此重叠。** 自动偏移后必须查看完整分辨率 PNG，逐项检查文字碰撞；发现重叠时只移动文字，不移动点位、峰位或河线。
- 每幅图应保留可复用配置、数据处理脚本、缓存和自动测试；新制图产物只输出 PNG。数据来源、行政区、主要地名和处理口径写入脚本注释、配置或配套笔记，便于检索与验证。
- **水系资产目录规范**：`OneNote/户外旅行/水系/` 根目录直下只保留 `.md` 笔记；PNG、JPG、PDF 等图像和导出产物统一放入 `OneNote/户外旅行/水系/scripts/outputs/`。制图脚本必须直接输出到 `scripts/outputs/`，不得在水系根目录生成重复副本。Obsidian 笔记统一使用 `![[scripts/outputs/文件名]]` 嵌入图片。缓存、下载数据和测试产物继续放在各自脚本目录下。
- 完成前必须验证：河名落在线上；干流和一级支流拓扑只有合理的下游出口；源头至汇入口连续；相邻流域干流已按范围和关联性补出；省界有独立颜色且无裁剪框伪边；涉及跨境区域时国境线与省界线型/颜色明确区分并进入图例；线宽随 `ORD_STRA` 单调递增；与流域相交的地级市无遗漏；三级城镇与关键地理节点符号互不混淆；乡镇未冒充县城；所有点位可见且文字未远离点位；地名、山名与河名无重叠；卫星底图 zoom 与流域面积匹配且原生拼图宽度达到清晰度门槛；PNG 宽度大于 3000 像素；水系根目录无图片残留。
