# 登顶山峰打卡地图制作规则

## 模式选择

当前有两条成熟实现线：

- **省域 DEM / 高程渲染模式**：广东登顶山峰打卡地图。底图为 DEM 分层设色 + 晕渲，叠加 Natural Earth 10m land/海岸线/河流、省界省名、城市、山脉和山峰标签。适合全省或大范围山峰打卡总览。
- **山脉卫星底图模式**：苍山十九峰打卡地图。底图为 Esri World Imagery，叠加 OSM `natural=peak` 峰点、已打卡状态、KML 轨迹和主脊关系。适合单条山脉或小范围路线/打卡状态展示。

如果用户没有指定模式，先问清楚使用哪一种；在用户确认前不能直接制作、下载数据或渲染。不要把省域 DEM 版和山脉卫星版混为一个视觉体系。

## 登顶山峰打卡地图制作规则（DEM 地形底图）

将 `户外运动经历/户外运动记录.md` 里的登顶山峰在省域 DEM 地形图上可视化。首个成品为广东省，设计已定型，可复制到其他省份。

### 脚本与产物位置

- 产物：`OneNote/户外旅行/户外运动经历/附件/山峰打卡地图/广东-登顶山峰打卡地图.png`
- 脚本目录：`…/山峰打卡地图/脚本/`
  - `peaks_gd_final.py`（山峰数据：名称/经纬度/海拔/来源）、`fetch_gd_dem.py`（下载省域 DEM→elev.npy）、`render_gd_peaks.py`（渲染）、`elev.npy`+`dem_meta.txt`（DEM 缓存）。
- 底图渲染逻辑复用自水系工具 `水系/scripts/terrain_dem/build_relief.py`（分层设色+晕渲）。
- 海陆边界：海洋不得再用 `DEM < 0` 或 `DEM < -50` 之类高程阈值判定；珠江口、香港周边浅海会失真。应叠加 Natural Earth 10m `ne_10m_land.geojson` 生成陆地面遮罩，陆地外统一涂海蓝，并叠加 `ne_coastline.geojson` 海岸线。
- 水系参考层：可叠加 Natural Earth 10m `ne_rivers.geojson`，只用一份 10m 河流数据，按 `scalerank` 控制线宽；不要同时叠加 50m/scale 版本，避免大河双线。DEM 地形底图上河流应使用浅色描边 + 蓝色主线，标注河名必须和实际叠加的 NE 河线匹配。
- 省界与省名：参考水系图模式 B，可复用本地 OSM admin1 或阿里云 DataV 省界缓存；省界使用稍宽紫色虚线，与河流、山峰和地形晕渲区分；省名用半透明浅灰紫大字，必须完整落在图幅内，不得被边框裁切。

### 制作流程（复制到其他省份时）

1. 从户外记录提取该省登顶山峰（含"环线/徒步"记录，用户明确这些视为登顶）。
2. 用 OSM/Nominatim 查证坐标海拔（见 CLAUDE.md 第5条查询规则）。OSM `natural=peak` 有 `ele` 时优先采用 OSM `ele`；OSM 有峰点但无 `ele` 时，用 NASA SRTM HGT、Copernicus DEM、AWS Terrain 等高程/等高线数据在峰点附近小窗口取局部最大值估算，标注加 `~`，并在 `src`/`note` 中记录数据源、窗口口径和查询结果。不得把 OSM 主库中不存在的等高线说成 OSM 等高线。
3. 改 `fetch_gd_dem.py` 的 bbox 为目标省范围，下载 DEM。
4. 改 `render_*.py` 的城市/山脉/河流列表为该省，渲染。

### 影响最终效果的关键要点

1. **底图分层设色降饱和 + sqrt 拉伸**：`norm=power(norm,0.7)`，色带用浅绿系（不用高饱和纯绿），晕渲 `vert_exag=3.0`、`hs=0.42+0.58*hs`，否则平原一片浓绿、地形起伏看不出。海洋必须用 Natural Earth 10m land 面遮罩判定并涂蓝，再叠加 10m 海岸线；不要用 DEM 负值阈值判海，珠江口和香港浅海会错。
2. **山峰按海拔三色三角**：高(≥1500m 深红)/中(800-1500m 橙)/低(<800m 绿)，点大小也随海拔。
3. **标签防重叠必做**：用真实文字包围盒做避让。硬约束包括：山峰名/海拔文字之间不得重叠，文字不得与山峰三角图标重叠，文字不得与城市、山脉、河名、省名、图例等固定标注重叠；图例区要预留为禁放框。密集山峰（如黄梅四峰、深圳大鹏群峰）不得靠缩小字号解决，字号保持可读；应把标签列到邻近空白区，用白底深灰虚线或同等清晰的引导线连接真实峰点。引导线可以拉开密集标签，但文字不应无约束地离山峰过远；若标签被甩远，必须重新检查避让候选和密集组布局。改完必须查看完整分辨率 PNG，必要时裁局部复查。
4. **DEM/SRTM 海拔估算的局限**：省域渲染缓存 zoom9 约 300m/格，只适合底图，不适合最终山峰估高；山峰坐标若落在山脚会偏低（如茶山顶旧估值明显偏低）。缺 OSM `ele` 时应优先用 NASA SRTM HGT 或同级高程数据，在 300-800m 半径小窗口取局部最大值估算；估算值标 `~` 并在图例注明，不谎称精确。
5. **坐标查证严守 CLAUDE.md 第5条**：纯名搜、多同名用地市过滤结果、别名（石猪山=猪蹄石）、非 peak 类型（大东山=memorial）需特殊处理，记录 source。
6. **城市标签越界**：城市点+标签须完整落在图幅内（留边距）才画，否则标签跑到图框外。
7. 标题副标题含统计：总数 + 高/中/低分级计数 + 最高峰。


## 卫星底图模式（苍山十九峰打卡图）

参考实现：`OneNote/户外旅行/户外运动经历/附件/苍山山峰打卡地图/脚本/make_cangshan_peak_checkin_map.py`。

- 产物：`OneNote/户外旅行/户外运动经历/附件/苍山山峰打卡地图/苍山山峰打卡图_卫星底图.png`；在 `户外运动记录.md` 中用 `![[附件/苍山山峰打卡地图/苍山山峰打卡图_卫星底图.png]]` 嵌入。
- 底图：Esri World Imagery，WGS-84 / Web Mercator tiles；苍山脚本当前 `SAT_ZOOM = 14`，`DISPLAY_BOUNDS = (99.94, 25.555, 100.37, 25.985)`。
- 高程辅助：AWS Terrain Tiles / Terrarium，苍山脚本当前 `DEM_ZOOM = 12`，用于主脊和局部地形辅助，不替代 OSM 峰点身份。
- 峰点：优先使用 OSM `natural=peak` 节点，峰名、经纬度、`ele`、OSM ID 必须保留；已打卡状态由户外运动记录和脚本中的 `CHECKED_OSM_IDS` 控制。
- 轨迹：读取 `附件/苍山运动轨迹/*.kml`；同一年轨迹使用同一种颜色，年份颜色在 `TRACK_COLORS` 中集中配置。轨迹只作行走记录层，不可用来移动峰点。
- 主脊：使用经核实的 OSM 峰点 ID 序列作为控制锚点；卫星打卡图可简化表达主脊关系，但峰点仍必须落在真实坐标上。
- 标签：峰点不移动；标签通过 `LABEL_OFFSETS` 和引导线/描边控制可读性。新增峰点或改动标签后，必须检查文字不重叠、不压住峰点、不遮挡关键轨迹。
- 验收：运行脚本重新生成 PNG；核对打卡数量、峰点是否在苍山主脊附近、轨迹年份颜色是否分明、底图是否清晰、标签是否可读。
