# Water System Common

## 水系图制作模式选择（先问后做）

目前有三套成熟的水系图制作流程，**开始制作前必须先询问用户使用哪种模式**，不得默认，也不得在用户选择前下载、处理数据或渲染成品：

- **模式 A · 卫星底图**（HydroBASINS + HydroRIVERS + OSM + 卫星影像）：底图为真实卫星影像，写实、纹理丰富，适合展现地表覆盖、城市与河谷实景。规范见下文《水系图制作规则（卫星底图模式）》。
- **模式 B · 地形分层设色**（DEM 晕渲 + pysheds 水文分析 + OSM 注记）：底图为高程分层设色（绿→黄→橙）加山体阴影，清晰突出地形骨架、分水岭与流域范围，风格接近传统地理教学挂图。规范见下文《水系图制作规则（地形分层设色模式）》。
- **模式 C · 卫星底图 + DEM 河网**（Esri 卫星影像 + AWS Terrain DEM + pysheds 水文分析）：结合 A、B 优点——底图用 Esri 卫星实景（保持明亮自然、**不加流域聚光灯**，否则周围过黑），河网、流域界、干流全部从 DEM 流向场提取（天然拓扑连通，无断口无毛刺，线宽随集水面积连续变化），标注体系默认采用模式 B 的浅色地形图方案（元江实践证明优于模式 A 暗底标注），配经纬网、比例尺、指北针。首个成品为汉江，后续默认以元江的模式 B 标注方案为准。规范见下文《水系图制作规则（卫星底图 + DEM 河网模式）》。

三种模式的通用规则（河网连续性优先、城镇分级、山脉山峰标注、文字不重叠、注明来源与免责声明、PNG 输出等）一致，仅底图与河网数据管线不同。**山脉位置、山峰位置和国境线表达属于三种模式共同遵守的硬约束**：山峰坐标、海拔和名称必须优先从 OpenStreetMap `natural=peak` 节点、地方政府/自然保护地/地名志等公开资料查询获得；山脉标注控制线必须依据 OSM 山系/山峰点、DEM 分水岭、地方政府地理描述或可靠地图资料综合校核；凡图幅涉及国境线，必须使用 OSM `admin_level=2` 国界关系或 Natural Earth `admin_0_boundary_lines_land` 等专门国界数据，用不同于省界/市界/流域界的独立线型和颜色标出，并在图例和配套说明中写明资料来源与口径。不得仅凭卫星底图观感、河谷位置、省级行政区外缘或版面空白手工估计地理对象位置；手工只允许调整文字偏移、字号和排版，不得作为峰位、山系位置、河名身份或国境线位置的依据。
