# Water System Mode C Satellite Dem

## 水系图制作规则（卫星底图 + DEM 河网模式，模式 C）

结合模式 A 的写实卫星底图与模式 B 的 DEM 水文河网。参考实现：汉江 `OneNote/户外旅行/水系/scripts/hanshui/`（`hanshui_config.py` + `prepare_hanshui_hydro.py` + `make_hanshui_basin_map.py`），运行环境复用 `terrain_dem/venv`。

- **数据分工**：底图用 Esri World Imagery（WGS-84）；高程用 AWS Terrain Tiles（Terrarium PNG）；流域界、河网、干流全部由 pysheds D8 水文分析从 DEM 提取（继承模式 B 的核心优点：河网与流向场天然拓扑一致，**结构上不可能出现毛刺、断口、支流不连干流**）；OSM 和公开资料用于河名、命名支流种子、标签定位校核与相邻水系干流（如长江）几何；省界用阿里云 DataV。
- **河名与命名支流通道规范**：模式 C 虽使用卫星底图，但支流名称身份规则与模式 B 相同。所有重要支流名称必须先由 OSM 同名 `waterway=*`、地方政府/水利部门资料、地名志、百科或可靠地图资料确认；种子点应取自 OSM 同名河道几何的上游端或中游稳定段，或公开资料明确的源头、流经节点、汇入口控制点。不得凭卫星底图观感、河谷形态或排版空白手工估计种子；配置中必须记录来源、查询日期、置信度和必要控制点。生成 DEM 河网后，应从核实种子沿 D8 下行追踪到目标干流，保存专属命名通道；标签只能吸附到该通道或经核实的平原 OSM 替换干流段。若 DEM 河网未能表达真实河道（如平原分流、界河、小河阈值过高导致缺失），应降低阈值、局部提高 DEM/河网精度、采用经说明的 OSM 订正段，或暂不标注该河名，不能把名称吸附到附近无身份河线。
- **坐标对齐关键设计**：Terrarium DEM 瓦片与 Esri 影像瓦片是**同一张 Web-Mercator 瓦片金字塔**——两者取同一 zoom、同一瓦片范围时逐像素 1:1 对齐。因此全图统一在**全局 Mercator 像素空间**绘制：流域掩膜/河网栅格坐标直接落在影像像素上；pysheds 输出的"线性纬度"坐标必须先逆变换回栅格行列、再转像素（`lin2px`），不得当作真实纬度直接投影（等距纬度与 Mercator 行在大纬度跨度下差可达数公里）。OSM/城市等真实 WGS-84 坐标用标准 Mercator 公式（`ll2px`）转像素。像素空间共形，河名旋转角无需纬度校正。
- **zoom 选择**：DEM 与影像通常同 zoom。流域数万 km² 以上用 z9（约 300 m/格，水文网格数百万格，pysheds+numba 一两分钟）；1–5 万 km² 用 z10；更小流域 z11。影像拼图仍须满足卫星模式的清晰度门槛（横向原生 ≥2,500 px 且 ≥ 最终 PNG 宽的 65%）。
- **出水口与流域校验**：出水口选在干流汇入更大水系之汇口上游足够远处（汉江选蔡甸附近、离长江约 30 km），防止 `snap_to_mask` 吸附到相邻大江；catchment 面积须与官方值比对（容差约 ±20–25%，DEM 分辨率会略偏大），超窗即报错停止。
- **干流追踪与公认河源（含 V 形陷阱）**：从出水口沿流向场逐格向上游取最大累积路径，得到干流。最大累积追踪是单调上溯，天然不产生折返。**默认直接采用这个自然追踪结果，不要为了联通某个"源头山峰"而强行改接**——山峰点若不在最大累积路径上（如汉江嶓冢山在中源漾水、并非最大头支），强行播种拼接会在接缝处产生不自然的 V 形北折/回头弯。
  - 若确需让干流走**公认正源**（如汉江官方正源是南源玉带河，而 DEM 最大累积走的是北源），做法是把追踪的**源头种子点设到正源的真实河源最上游端**（config `SOURCE_SEED`，如玉带河源 106.163,32.767），从该种子沿 D8 下行、遇原干流即汇（应落在正源汇入干流的节点附近，如勉县 `SOURCE_JUNCTION`），拼接为「出水口→汇点→正源源头」。**种子必须是河源最上游的河道格点，不能是河道旁的山坡**——种在坡上 D8 会先下坡到河道再上行，仍出 V。全程是 DEM 几何，只有种子坐标来自正源位置，符合"源头段不用 OSM 线替换"。
  - 北盘江能顺利重建是因为马雄山恰在最大累积路径上；汉江嶓冢山不在，故改用玉带河正源种子。见汉江 `prepare_hanshui_hydro.py`（`SOURCE_SEED`/`SOURCE_JUNCTION` + ring 搜索最长汇入路径）。
- **河网连续渲染与阈值**：`extract_river_network` 提取，每支附 `acc_km2`，线宽按集水面积对数**连续**映射，避免离散阈值导致突变。模式 C 默认改用模式 B 的浅色地形图河网视觉：支流统一浅蓝色相（参考元江/湘江 `#56B4E9`），干流与支流使用同一流量—线宽函数，不得另加固定粗线；若需突出干流，只能保持色相和饱和度不变、降低 HSL 明度（参考 `#12618D`），并在干流周围铺设可辨认但边缘柔和的白色高斯渐变河谷背景。轻度移动平均平滑消除栅格锯齿。**起绘阈值 `NET_MIN_KM2` 要按流域规模和观感调**：太低（如汉江初版 250 km²、385 条）毛细支流过密显凌乱，大流域建议 800 km²+；图例文字用阈值动态生成避免不同步。**严禁把背景 DEM 河网与 OSM 河流几何叠画**（干流的平原订正见下条，是整段替换而非叠加）。
- **平原/三角洲段干流用 OSM 真实河道订正**：DEM 的 D8 在地势平坦、河道受人工堤防约束的平原/三角洲会**误走分流**（汉江被 D8 送去天门河，而非真实的沙洋—潜江—仙桃—汉川堤防河道）。修法：山区河段保留 DEM 干流（准确且贴合地形），平原段整段替换为**权威 OSM 干流河道**——取 OSM 同名干流（`waterway=river` name 匹配）`linemerge` 后选"最接近河口的连通段"、定向为上游→河口，在拼接点（config `LOWER_SPLICE_HINT`，汉江≈宜城）把 DEM 干流上游段与 OSM 下游段首尾相接。这是**干流单线的整段替换**，不违反"禁止叠画背景河网"。见汉江 `prepare_hanshui_hydro.py` 的 `fetch_osm_lower_mainstem()`；图例干流标注为"山区 DEM／平原 OSM"。
- **底图保持明亮、不加聚光灯**：卫星底图是模式 C 的主体观感，须保持明亮自然（仅做极轻微 Color/Contrast/Brightness ≈1.0–1.08 微调，切勿像模式 A 那样压暗到 0.56）。**不要在流域外叠加聚光灯暗蒙版**——那会让周围地形全黑、失去卫星实景的意义（2026-07-06 实测：加聚光灯后四周太黑，已取消）。流域范围仅靠边界线勾勒即可：流域界用浅青白 `#D5F3EA` 实线（不用模式 B 的红线，避免与青色河网、暖橙省界撞色）；因底图明亮，所有文字须带深色描边（`#0A1C18`）保证可读。
- **标注体系默认采用模式 B 方案**：模式 C 不再默认沿用模式 A 暗底亮河配色；除单图明确说明外，应采用元江验证后的模式 B 标注体系。河名使用模式 B 的蓝色层级和描边，必须绑定专属命名通道并沿河线旋转；干流名可用更大字号/字重突出，但不能脱离河线。城市与县城采用模式 B 的分级点符号与字号体系；山脉名、山峰三角与海拔标注沿用模式 B/通用规则，山峰坐标须 OSM 或公开资料校核；水库、水电站、源头、汇入口、河口等地理节点按模式 B 的符号层级处理。相邻水系干流以低于目标水系的浅蓝/灰蓝层级表示。所有文字必须带适合卫星底图的浅/深描边以保证可读，但不得使用大面积暗底或流域外聚光灯来服务标注。
- **版式**：带细经纬网（低透明度白线+边缘度标）、比例尺（按 Mercator 逐纬度换算 m/px）、指北针、图例、标题/副标题（副标题写明源头—干流节点—河口与 DEM 提取面积）、来源与免责声明；只输出 PNG（宽 >3,000 px）。
- **通用规则适用**：文字互不重叠、拥挤处删次要标注、成品与脚本/缓存分离（成品只进 `scripts/outputs/`）、配套同名水系介绍笔记。
